# Emocre Site — Marketing & Creature Compendium
Static site for Emocre, built with **Eleventy 3 + SCSS (Dart Sass)**. Served at the domain in `CNAME` (`emocre.com`). It showcases the creature roster (data + art) and the press kit, all sourced from Notion and the `emocre-art` repo — nothing about creatures is authored here by hand.

## Critical Rules

1. **NEVER edit generated/imported content** — re-run the step that produces it:
   - `data/` — imported from Notion by `make import`. **gitignored.**
   - `_data/creatures.json` — produced by `make transpile` (`tools/transpile.ts`). Edit the transpiler, not the JSON.
   - `assets/creatures/*` — creature images copied by `make transpile` from the pre-rendered CRT sprites (`*-art-front-crt.png`) generated in emocre-art.
   - `pt/press-kit.md` — overwritten wholesale (front matter included) by `make import`. Edit `en/press-kit.md` freely (not touched by import); for `pt/press-kit.md` edit the Notion press kit page instead.
   - `_site/` — Eleventy build output. **gitignored.**
2. **NEVER edit `vendor/`** — `vendor/emocre-art` and `vendor/emocre-tools` are cloned by `make install` and **gitignored**. Fix shared code in the `emocre-tools` repo; fix art in `emocre-art`.
3. **Hand-authored sources are fair game:** `_data/site.json` (site config), `_data/i18n/*.json` (UI strings), `_includes/` (templates), `assets/css` (SCSS), `assets/js`, `index.html` (language router), `home.html`, `play.html`, `saudalos.html`, `en/press-kit.md`, `legacy-*.html` (old-URL redirect stubs).

## i18n (English site)

The site is bilingual: `/pt/...` and `/en/...`, generated from a single template per page via
Eleventy pagination over `_data/locales.json` (`["pt", "en"]`), with the pagination alias
`locale` available in the template. Root `index.html` is a browser-language redirect
(`layout: null`); `legacy-press-kit.html` / `legacy-play.html` / `legacy-saudalos.html` are
redirect stubs for the pre-i18n flat URLs (`/press-kit`, `/play`, `/saudalos`), pointing to
their `/pt/...` equivalent.

UI copy lives in `_data/i18n/pt.json` / `_data/i18n/en.json` (Eleventy auto-namespaces these
as global `i18n.pt` / `i18n.en`). Templates read `{{ i18n[locale].section.key }}`; inline
`<script>` blocks get `const LOCALE = "{{ locale }}"` and `const I18N = {{ i18n[locale] | jsonify }}`
injected so client-side JS (creature cards, modal) can pick the right string/field too.
Creature-level English fields (`nameEn`, `descriptionEn`, `type1En`, `type2En`) come from
`tools/transpile.ts`, falling back to the PT value when the Notion `_en` field is empty.

`pt/press-kit.md` and `en/press-kit.md` are full separate markdown files (not i18n-JSON
driven, since it's article-length content) — each directory has a small Eleventy directory
data file (`pt/pt.json` / `en/en.json`) that sets `locale` for anything under it, since
`pt/press-kit.md`'s own front matter gets replaced wholesale by `make import`.

## Data Workflow

```
Notion ──make import──▶ data/ (YAML) ─┐
emocre-art (vendor) art/ ─────────────┴─make transpile──▶ _data/creatures.json + assets/creatures/*
                                                          └─make build / start──▶ _site/ (Eleventy)
```

`make transpile` (`tools/transpile.ts`) reads the type chart from `data/`, pulls creatures flagged `site` from the vendored `art/` via `@emocre/tools`'s `CreaturesFileRepository`, computes type weaknesses, copies the pre-rendered CRT front sprite (`*-art-front-crt.png`, generated in emocre-art) into `assets/creatures/`, and writes `_data/creatures.json`.

> Because transpile reads `vendor/emocre-art/art/`, the site depends on `emocre-art` exposing an `art/` directory (pulled via `git sparse-checkout set art`). If that repo's layout changes, `make install` and the transpiler must be updated to match.

## Commands

```bash
make install      # Clone/update vendor (emocre-tools + sparse art) + yarn install
make import       # Pull creatures/types + press-kit from Notion into data/
make transpile    # Build _data/creatures.json + assets/creatures from data/ + vendored art
make build        # Eleventy production build → _site/
make start        # install + import + transpile + Eleventy dev server (watch)
yarn build / yarn start   # Eleventy only (no data refresh)
```

## Code Conventions

- TypeScript tooling runs via `ts-node` (`transpileOnly`, `strict: false`); `camelCase`/`PascalCase`.
- Emocre-specific paths live in `tools/paths.ts` — keep path knowledge there, not scattered through the transpiler.
- Eleventy formats: `scss`, `html`, `md` (see `package.json` scripts).
- UI copy defaults to **pt-BR** (see `_data/site.json`) with an English translation in `_data/i18n/en.json`; creature names/descriptions come from Notion in PT, with English fields falling back to PT when untranslated.

## Environment

`.env`: `NOTION_API_KEY=<key>` (consumed by `make import` via `@emocre/tools`).
